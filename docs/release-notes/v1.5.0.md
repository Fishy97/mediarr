# Mediarr 1.5.0 Release Notes

Mediarr 1.5.0 adds live Jellyfin production acceptance tooling and strengthens activity recommendations for series and anime-style libraries.

## Highlights

- Adds an opt-in live Jellyfin acceptance suite for real NAS libraries.
- Generates local JSON and Markdown reports with import counts, file-size totals, local verification coverage, unmapped paths, activity rollups, warnings, and top suggestions.
- Adds in-app ingestion diagnostics and `GET /api/v1/integrations/{id}/diagnostics` for persisted media-server proof summaries.
- Keeps acceptance runs isolated from normal `/config/mediarr.db` state by using a scratch database under the ignored report directory.
- Adds series-level activity recommendations by aggregating episode files into inactive or abandoned series suggestions.
- Treats anime libraries as series-style recommendations while preserving a clear `anime` evidence category when library/path context identifies it.
- Suppresses active series when recently added episodes indicate the show may still be current.

## Safety Notes

The live acceptance suite is read-only. It does not refresh Jellyfin libraries, write Jellyfin state, or modify media files. Reports are written under `acceptance-reports/`, which is ignored by git.
