# Mediarr 1.3.0 Release Notes

Mediarr 1.3.0 closes the release-hardening gaps after the premium stewardship release. The focus is safer public self-hosting, stronger media-server parity, and clearer operational guarantees.

## Highlights

- Emby is now a first-class ingestion source alongside Jellyfin and Plex.
- Plex watch-history sync stores and reuses a cursor while preserving prior normalized activity rollups.
- Media-server and provider requests retry bounded `429` and `5xx` failures with `Retry-After` awareness.
- Integration cards expose the retry/backoff posture in the MediaGlass UI.
- Admin sessions prune expired rows and set Secure cookies when Mediarr is reached through HTTPS or an HTTPS reverse proxy.
- CI now includes a no-delete safety gate that scans for destructive media operations and verifies read-only media mounts.
- Tagged GHCR image releases generate signed GitHub artifact provenance attestations.
- A formal threat model documents trust boundaries, risks, non-goals, and release verification expectations.

## Safety Notes

Mediarr remains suggest-only. There is still no permanent media deletion route, and media mounts remain read-only by default in Docker Compose.

## Validation Gate

Release validation requires:

- `go test ./...`
- `go vet ./...`
- `npm --prefix frontend run test -- --run`
- `npm --prefix frontend run build`
- `scripts/verify-no-delete.sh`
- `docker compose config --quiet`
- `docker compose --profile ai config --quiet`
- `docker compose build mediarr`
