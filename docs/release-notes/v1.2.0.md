# Mediarr 1.2.0 Release Notes

Mediarr 1.2.0 focuses on premium, evidence-first stewardship. The release keeps Mediarr suggest-only while making recommendations more transparent, safer to review, and easier to trust on large Jellyfin and Plex libraries.

## Highlights

- Recommendation proof endpoint with storage verification, activity evidence, source rule, risk, and AI attribution fields
- Recommendation states for new, reviewing, ignored, protected, and accepted-for-manual-action review items
- Path mapping workbench for server paths versus Mediarr-visible paths
- Mapping verification that upgrades evidence to `local_verified` when local files exist and sizes match
- Unmapped media-server items stay in a review queue instead of becoming cleanup recommendations
- Background job cancel, retry, and stale recovery support
- MediaGlass UI updates for proof cards, mapping review, and progress controls
- GHCR image publishing workflow for tagged releases

## Safety Posture

Mediarr still does not provide a permanent media deletion endpoint. Cleanup remains advisory. Protected recommendations leave the open queue, accepted recommendations remain visible for manual action, and all state transitions are audit logged.

## Validation Gate

Release validation requires backend tests and vet, frontend tests and build, Docker Compose no-AI and AI config checks, Docker image build, and smoke testing `/api/v1/health` plus first-run setup state.
