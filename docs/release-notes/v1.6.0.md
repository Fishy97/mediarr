# Mediarr 1.6.0 Release Notes

Mediarr 1.6.0 adds the Stewardship Advantage phase: request intent, analytics enrichment, safer campaign planning, and dashboard proof that separates verified storage savings from estimates.

## Highlights

- Adds Seerr-compatible request ingestion for Jellyseerr and Overseerr style `/api/v1/request` payloads.
- Adds optional Tautulli activity enrichment for Plex after Plex inventory has been synced.
- Adds built-in campaign templates for cold movies, abandoned series, anime backlog, verified high-storage items, never-watched large files, and requested-but-never-watched media.
- Adds campaign what-if simulation with request conflicts, protection conflicts, unmapped blockers, estimated savings, and verified savings.
- Adds dry-run Leaving Soon collection publication plans that show publishable and blocked items before any collection write. Jellyfin collection publishing is supported; Plex remains preview-only in this release.
- Adds a storage ledger that separates locally verified bytes, mapped estimates, server-reported estimates, unmapped blockers, protected bytes, accepted manual action, and requested media.
- Adds dashboard notifications and protection request review/approval.
- Adds REST routes and frontend controls for all stewardship signal sources and safety workflows.

## Safety Notes

This release remains suggest-only. It does not delete, move, quarantine, unmonitor, clear requests, or change media-server watch history.

Leaving Soon publication is collection-only. Preview is the normal workflow, and Jellyfin publish requires explicit confirmation. Candidates without external item IDs or enough path evidence are blocked in the plan instead of silently included.

Seerr and Tautulli syncs store normalized signal fields only. API keys are stored in `/config/mediarr.db` and returned to the browser only as redacted configuration status.
